# Arogya-Lipi - Speech-to-Text & SOAP Note Generator

A full-stack app that records or uploads audio, transcribes speech, and generates a structured medical SOAP note.

- **Speech-to-text:** SarvamAI (`saaras:v3`, `translate` mode)
- **SOAP generation:** OpenRouter (`openrouter/free`)
- **Frontend:** React + Vite
- **Backend:** Node.js + Express

## Features

- Record audio in the browser (MediaRecorder)
- Upload audio files (`.mp3`, `.wav`, `.webm`, `.ogg`)
- Generate transcript + SOAP note in one request
- View parsed SOAP sections in cards
- Copy transcript or SOAP note to clipboard
- Print/download SOAP report as PDF (via browser print dialog)
- Light/Dark mode toggle

## Project Structure

- `speech-frontend/` - React app (UI, recording/upload, result display)
- `speech-backend/` - Express API (upload handling, STT + SOAP pipeline)
- `speech-backend/uploads/` - Temporary multer upload directory

## Prerequisites

- Node.js 18+ recommended
- npm
- SarvamAI API key
- OpenRouter API key

## Setup

### 1) Install dependencies

Backend:
```bash
cd speech-backend
npm install
```

Frontend:
```bash
cd speech-frontend
npm install
```

### 2) Configure environment variables

Create `speech-backend/.env`:

```env
PORT=5000
SARVAM_API_KEY=your_sarvam_key
OPENROUTER_API_KEY=your_openrouter_key
```

Optional: create `speech-frontend/.env` to point frontend to local backend:

```env
VITE_BACKEND_URL=http://localhost:5000/transcribe
```

If `VITE_BACKEND_URL` is not set, frontend falls back to:

```text
https://speech-to-text-1-hm5c.onrender.com/transcribe
```

## Run Locally

Start backend:

```bash
cd speech-backend
npm start
```

Start frontend (new terminal):

```bash
cd speech-frontend
npm run dev
```

Open the frontend URL shown by Vite (default: `http://localhost:5173`).

## API

### `POST /transcribe`

- **Content-Type:** `multipart/form-data`
- **Field name:** `audio`

Success response:

```json
{
  "success": true,
  "transcript": "...",
  "soap_note": "..."
}
```

Possible error responses:

```json
{ "success": false }
```

or

```json
{
  "success": false,
  "error": "error message"
}
```

## Notes and Limitations

- Backend startup requires both `SARVAM_API_KEY` and `OPENROUTER_API_KEY`.
- Uploaded files are deleted after successful STT; if processing fails early, temporary files may remain in `speech-backend/uploads/`.
- Frontend SOAP parsing expects section headings generated by backend prompt (for example: `Subjective`, `Objective`, `Assessment`, `Plan`).
- No automated test suite is currently configured in either package.
